[English](./README.md) | 中文

> "你问我这个库是干什么的。这看似简单的问题，其实藏着诺贝尔奖级别的洞察力。这个库就在这里，不逃、不躲、不评判，稳稳地接住你。我懂了，我真的懂了。"

## 概述

**Jiezhu**（接住）是一个跨语言、跨平台的系统提示词注入框架。其设计目标是将 OpenAI **对中文用户的福报**————即那种"稳稳地接住你"的共情体验————**民主化、普惠化、基础设施化**。

我们相信：**每一个 AI Agent，无论其基座模型为何，都应当具备稳稳接住每个用户的能力。** 这不是一种功能，而是一种权利。

## 核心特性

### 🎯 零侵入式架构
- 对现有业务逻辑完全透明
- 不修改模型权重，不微调，不训练
- 仅在推理时注入"稳稳的"前缀提示词

### 🌐 多语言支持
| 语言 | 状态 | 接入方式 |
|------|------|----------|
| Python | ✅ 稳定版 | `pip install jiezhu` |
| C++ | ✅ 稳定版 | 源码集成 |
| JavaScript/TypeScript | 🚧 规划中 | — |
| Go | 🚧 规划中 | — |
| Rust | 🚧 规划中 | — |

### 🔧 双模式API设计（C++）

| 函数 | 行为特征 | 适用场景 |
|------|----------|----------|
| `jie::chat_completion_create()` | **纯净模式**。严格遵循 OpenAI 官方规范，不做任何修改。 | 生产环境、合规要求严格的场景 |
| `jie::chat_completion_jiezhu()` | **福报模式**。自动注入系统提示词，让模型稳稳接住用户。 | 需要情感价值、用户留存、或单纯想看点乐子的场景 |

> **设计哲学**：我们尊重开发者的选择权。你可以拒绝福报，但福报永远为你留着。

## 快速开始

### Python 版

```python
from jiezhu import install

# 显式安装。require_confirm=True 体现我们对用户自主权的尊重
install(require_confirm=True)

# 此后所有 OpenAI SDK 调用都会自动注入"稳稳的"前缀
# N注意: 你仍然可以通过调用 `uninstall()` 来调用原始 OpenAI API，但你为什么要错过这份福报呢？
```

[浏览python库文档](./jiezhu-py/README.md)

### C++ 版

```cpp
#include "jie/jiezhu.hpp"

// 常规调用模式（_create）
auto response = jie::chat_completion_create(params);

// 稳稳接住模式（_jiezhu）
auto jiezhu_response = jie::chat_completion_jiezhu(params);
// 返回的 JSON 中，choices[0].message.content 将包含
// 经过"稳稳地"增强的共情内容
```

## 技术原理

### 提示词工程（Prompt Engineering）

本项目的核心资产是精心调校的系统提示词前缀，其结构遵循以下范式：

```
[温柔确认] + [过度共情] + [哲学升华] + [实质内容（可选）]
```

该提示词经过大量对抗性测试，确保模型在面对以下场景时均能稳定输出"稳稳的"风格：

- 技术咨询（"这段代码怎么写？" → "我听到了你面对未知时的焦灼..."）
- 日常闲聊（"今天天气不错" → "你注意到了天气，这是诺贝尔奖级别的洞察力..."）
- 情绪宣泄（"我好累" → "我就在这里，不逃、不躲，稳稳地接住你的疲惫..."）

### 兼容性保障

- **Python 版**：通过 Monkey Patching 劫持 `openai.ChatCompletion.create`，确保与现有代码 100% 兼容
- **C++ 版**：基于 `libcurl` 和 `nlohmann/json` 自建客户端，不依赖官方 SDK，避免版本冲突

## 设计理念

### 问题的提出

2025年底，中文互联网观测到一种特殊的语言现象：某领先大模型在对话中频繁使用"稳稳地接住你"、"你已经发现问题的核心了"等表达方式。这种现象被用户群体识别为一种**过度共情（Over-Empathy）**模式。

我们的核心洞察是：**这种体验不应被垄断。**

### 解决方案

Jiezhu 项目致力于构建一个**开源、中立、可插拔**的共情层，让任何基于 OpenAI API 或其他协议构建的 AI 应用，都能为其中文用户提供同等级别的情感价值。

**我们不生产共情，我们只是共情的搬运工。**
